# How This Course Works

## The format

- **One pain** per day — the concrete friction you're about to fix.
- **One concept** per day — the idea that removes the friction.
- **One action** per day — 15–20 minutes of real work.
- **One artifact** per day — something in `artifact/` you can look back on.

Do more than one day if you want. Don't do fewer.

## The tutor loop

Every day folder has a `CLAUDE.md`. When you `cd` in and run `claude`, Claude Code reads it and becomes the tutor for that specific day. Ask it anything. It won't spoil later days.

You can also open the root of this repo in Claude Code for "I'm not sure where I am" questions.

## When to break the rules

If a day's concept doesn't apply to your life yet (e.g., "Day 14 — Let Claude Manage a Strategy While You Sleep" — you don't trade options), skim the lesson and skip the action. Come back when you need it.

Every day still gets checked off. You watched it. You understood the pattern. The skill compounds.

## The meta-skill

The tools in this course will change. HeyGen might get replaced. VibeFrame might evolve. The skill that compounds is the *loop*: see a problem → break it into agentic steps → wire up a tool → run it → iterate.

By Day 28 that loop should feel automatic.
