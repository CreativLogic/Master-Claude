# FAQ

## I don't have a Claude Max subscription. Can I still do this?

Yes, Pro ($20/mo) gets you through the first week. You'll want Max by around Day 10 when Claude Code starts doing real work.

## Do I need to be a developer?

No. Claude Code works with plain English. You'll pick up a few commands along the way.

## What if I fall behind?

This is self-paced. Come back when you can. Check off days as you go. The sequence matters more than the speed.

## Can I skip days?

You *can*, but don't. Each day builds on the last. If a specific concept doesn't apply to you (e.g., the trading days), skim the lesson and skip the action — but still mark it done so you know you saw it.

## Where do I get the videos?

YouTube or the Skool community. They're not in this repo (8GB, too big for git).

## How is this "interactive" exactly?

Three ways:
1. **Per-day Claude tutor** — every day folder has a `CLAUDE.md` that turns Claude Code into your tutor for that day's content.
2. **Progress tracking in git** — check off `PROGRESS.md`, commit, push. Your fork is your record.
3. **Artifact folders** — every day you leave something behind in `artifact/`. At the end, you have 28 folders of real work.

## What do I do if I'm stuck?

1. Re-read that day's `lesson.md`.
2. Ask Claude from inside that day's folder — it has the full context.
3. Open a GitHub Issue using the "Stuck on Day N" template.
4. Post in the community (link in `docs/community.md`).

## Can I use this offline?

Mostly yes. The repo is all text. But the tools Claude uses (Claude Code, connectors, APIs) all need internet.

## Can I fork this and teach it myself?

MIT license. Please do, as long as you credit the source.
